# MMeAssocInd

## 协议章节

### 消息标识

- 管理消息名称：关联指示报文（`MMeAssocInd`）
- 管理消息类型：`0x0034`
- 来源文档：`doc/南网hplc+hrf双模模块2.0标准规范/通感一体化低压电力线宽带载波通信规约/4-通感一体化低压电力线宽带载波通信规约 第4部分 数据链路层通信协议 （初稿).pdf`
- 对应 PDF 物理页：50-51
- 对应正文页码：48-49

### 5.1.4.3 关联指示报文

关联指示报文（`MMeAssocInd`）格式定义如表 99 所示。

#### 表 99 关联指示报文格式

| 字段 | 字节号 | 字段大小（字节/比特） | 说明 |
| --- | --- | --- | --- |
| 结果 | 0 | 1 字节 | 见表 100 |
| 站点层级 | 1 | 1 字节 | 新入网站点所属的网络层级 |
| 站点 MAC 地址 | 2-7 | 6 字节 | 新入网站点的 MAC 地址 |
| CCO MAC 地址 | 8-13 | 6 字节 | 本网络的 CCO MAC 地址 |
| 站点 TEI | 14-15 | 2 字节 | CCO 为 STA 分配的设备标识 |
| 代理 TEI | 16-17 | 2 字节 | 为新入网 STA 分配的代理站点 TEI |
| 链路类型 | 18 | bit0 | 0 表示高速载波链路，1 表示无线链路 |
| 载波频段 | 18 | bit1-bit3 | 频段取值沿用关联确认报文定义 |
| 保留 | 18 | bit4-bit7 | 保留 |
| 保留 | 19-20 | 2 字节 | 保留 |
| 分包序号 | 21 | 1 字节 | 首个分包序号为 1 |
| 总分包数 | 22 | 1 字节 | 关联指示报文总分包数 |
| 最后一个分包标识 | 23 | 1 字节 | 见表 101 |
| 站点关联随机数 | 24-27 | 4 字节 | 原始关联请求中的随机数 |
| 保留 | 28-44 | 17 字节 | 保留 |
| 组网序列号 | 45 | 1 字节 | 关联确认所回应请求中的组网序列号 |
| 保留 | 46-47 | 2 字节 | 保留 |
| 重新关联时间 | 48-51 | 4 字节 | 关联被拒绝后允许重试的时间间隔，单位毫秒 |
| 端到端序列号 | 52-55 | 4 字节 | 原始关联请求中的端到端管理消息序列号 |
| 保留 | 56-63 | 8 字节 | 保留 |
| 路由表信息 | 64 起 | 可变长 | 参照 5.1.4.2.18 |

关联指示报文的 MAC 帧目的 TEI 必须填写目的站点 TEI。未入网 STA 需要处理目的 MAC 地址与其一致的单播关联指示报文。

### 表 100 关联请求的确认结果

| 值 | 定义 |
| --- | --- |
| 0x00 | 表示关联请求成功 |
| 0x01 | 表示该站点不在白名单中 |
| 0x02 | 保留 |
| 0x03 | 表示加入的站点个数超过上限 |
| 0x04 | 表示没有设置白名单列表 |
| 0x05 | 表示代理站点个数超过上限 |
| 0x06 | 表示子站点个数超过上限 |
| 0x07 | 表示没有回复 |
| 0x08 | 表示重复的 MAC 地址 |
| 0x09 | 表示超过拓扑层级 |
| 0x0A | 表示曾经入网的站点再次入网 |
| 0x0B | 表示新的站点试图以自己的子站点为代理来入网 |
| 0x0C | 表示组网拓扑中存在环路 |
| 0x0D | 表示 CCO 端未知原因出错 |

### 表 101 最后一个分包标识

| 值 | 定义 |
| --- | --- |
| 0x00 | 不是最后一个分包 |
| 0x01 | 是最后一个分包 |

## 实现说明

- `03_MMeAssocInd.h/.c` 完整实现了表 99 固定头和路由表信息头的编解码。
- `路由表信息` 复用 `5.1.4.2.18` 的结构，拆为头字段和原始 `RouteTable` 字节流，不引入动态内存。
- 解码函数不会拷贝 `RouteTable`，`Message->RouteTableInfo.RouteTable` 直接指向输入缓冲区中的对应位置。
- 保留字段统一要求为 0，便于在 demo 中尽早发现不符合规范的输入。

## 与 1.0 版本对比

对比基准为 `doc/南网hplc+hrf双模模块1.0标准规范/2、计量自动化系统双模通信技术规范/《南方电网计量自动化系统双模通信技术规范 第2部分：无线通信接入协议》（报批稿）.pdf` 的 `8.1.4.3 关联指示报文`（表 `177` 至表 `179`）。

从报文整体布局看，`2.0` 版 `MMeAssocInd` 仍延续 `1.0` 的固定 `64` 字节头部加可变长 `路由表信息` 结构，`结果`、`站点层级`、`MAC/TEI`、分包控制、`随机数`、`组网序列号`、`重新关联时间`、`端到端序列号` 的位置均保持一致；当前可识别的实质变化主要集中在字节 `18` 的 `载波频段` 位域扩展。

| 对比项 | 1.0 版 | 2.0 版 | 结论 |
| --- | --- | --- | --- |
| 章节归属 | `8.1.4.3`，表 `177-179`，归属于 `1.0` 双模无线通信接入协议 | `5.1.4.3`，表 `99-101`，归属于 `2.0` 数据链路层通信协议 | 规范迁移并重编号 |
| 固定头与分包骨架 | 固定头为 `64` 字节，后接可变长 `路由表信息`；`分包序号`、`总分包数`、`最后一个分包标识` 位于字节 `21-23` | 与 `1.0` 相同，固定头长度和分包字段位置未变 | 外层布局未变 |
| 结果枚举 | 表 `178` 定义 `0x00-0x0D`，其中 `0x07` 为“表示没有回复” | 表 `100` 定义与 `1.0` 一致 | 关联结果语义未变 |
| 字节 `18` 位域 | `bit0` 为 `链路类型`，`bit1-bit2` 为 `载波频段`，`bit3-bit7` 保留 | 当前 `2.0` 文件按页图人工校准为 `bit0` 为 `链路类型`，`bit1-bit3` 为 `载波频段`，`bit4-bit7` 保留 | `2.0` 在现有实现口径下扩展了频段表达能力 |
| 路由表信息引用 | 参照 `8.1.4.2.18` 路由表信息 | 参照 `5.1.4.2.18` 路由表信息 | 仅章节号变化，复用结构未变 |

整体上，`2.0` 版 `MMeAssocInd` 仍属于在 `1.0` 关联指示报文基础上的兼容扩展，而不是报文重设计。

## 校准说明

- 原 PDF 为扫描版，表 99 OCR 误识别较多，本文件依据页面图像人工校准。
- 字节 21 和字节 22 已人工校准为“分包序号”与“总分包数”，不是反向顺序。
- 字节 18 已人工校准为 `bit0` 链路类型、`bit1-bit3` 载波频段、`bit4-bit7` 保留。
- 结果值 `0x07` 在表 100 中明确为“表示没有回复”，实现中按有效结果处理。
- 消息类型 `0x0034` 与 `h2-link-v2-sdk/SDK/src/spg/mac/mmsg.h` 中的 `MMeAssocInd` 定义一致。
