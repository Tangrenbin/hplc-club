# MMeRFDiscoverNodeList

## 协议章节

### 消息标识

- 管理消息名称：无线发现列表报文（`MMeRFDiscoverNodeList`）
- 管理消息类型：`0x0071`
- 来源文档：`doc/南网hplc+hrf双模模块2.0标准规范/通感一体化低压电力线宽带载波通信规约/4-通感一体化低压电力线宽带载波通信规约 第4部分 数据链路层通信协议 （初稿).pdf`
- 对应 PDF 物理页：63-66
- 对应正文页码：61-64
- 用户提到的“第 63 页”对应正文页码 `63`，实际是 PDF 物理页 `65`

### 5.1.4.18 无线发现列表报文

无线发现列表报文由固定头和原始信息单元区组成：

- 固定头 `7` 字节
  - 字节 `0-5`：站点 `MAC` 地址
  - 字节 `6`：统计序号
- 随后是 `N` 个信息单元，每个信息单元由：
  - `1` 字节头：`bit[6:0]` 为信息单元类型，`bit[7]` 为长度类型
  - `1` 或 `2` 字节长度字段
  - 可变长内容区

### 5.1.4.18.3 信息单元类型

- `0`：站点属性信息
- `1`：站点路由信息
- `2`：邻居节点信道信息非位图版
- `3`：邻居节点信道信息位图版
- 其他：保留

### 5.1.4.18.4 长度类型

- `0`：信息单元长度字段为 `1` 字节
- `1`：信息单元长度字段为 `2` 字节

### 5.1.4.18.6 站点属性信息

本次实现已对表 `142` 做结构化支持。结合下一页续表，内容区总长为 `14` 字节，字段顺序与 SDK 中 `sta_attr_info_t` 对齐：

- `CCO MAC` 地址 `6` 字节
- `代理 TEI` `12 bit`
- `角色` `4 bit`
- `层级` `4 bit`
- `链路 RF 跳数` `4 bit`
- `代理上行接收率` `1` 字节
- `代理下行接收率` `1` 字节
- `链路最小接收率` `1` 字节
- `无线发现列表周期` `1` 字节
- `无线接收率老化周期个数` `1` 字节

代码提供：

- `mme_rf_discover_node_list_station_attr_info_encode`
- `mme_rf_discover_node_list_station_attr_info_decode`

### 5.1.4.18.7 站点路由信息

本次实现已对表 `143` 做结构化支持：

- 每个路由条目固定 `2` 字节
- `12 bit` 下一跳站点 `TEI`
- `4 bit` 路由类型
- 路由类型沿用表 `125`

代码提供：

- `mme_rf_discover_node_list_route_entry_encode`
- `mme_rf_discover_node_list_route_entry_decode`
- `mme_rf_discover_node_list_route_info_get_entry_count`

### 5.1.4.18.8 与 5.1.4.18.9 邻居节点信道信息

正文第 `62-63` 页主要定义了两类复杂可变信息单元：

- 邻居节点信道信息非位图版
- 邻居节点信道信息位图版

这两类信息单元内部还继续嵌套：

- 组合类型
- 不同字段长度组合
- `SNR` 与 `RSSI` 量化映射
- 位图起始 `TEI` 与位图段

本次实现策略：

- 主报文和信息单元头完整编解码
- 信息单元类型 `2`、`3` 的内容区暂保留为原始字节区
- 不在当前版本继续拆分邻居节点信道组合字段

## 实现说明

- `18_MMeRFDiscoverNodeList.h/.c` 已实现主报文编解码，主结构保存原始信息单元区。
- `validate` 会顺序遍历原始信息单元区，确保每个信息单元头、长度和值域都自洽。
- 站点属性信息和站点路由信息提供了独立的辅助编解码接口。
- 邻居节点信道信息仅做头部级别校验，内容保留原始字节。

## 与 1.0 版本对比

对比基准为 `doc/南网hplc+hrf双模模块1.0标准规范/2、计量自动化系统双模通信技术规范/《南方电网计量自动化系统双模通信技术规范 第2部分：无线通信接入协议》（报批稿）.pdf` 的 `8.1.4.18 无线发现列表报文`（表 `214` 至表 `227`）。

从报文整体布局看，`2.0` 版 `MMeRFDiscoverNodeList` 延续了 `1.0` 的“`6` 字节站点 `MAC` 地址 + `1` 字节统计序号 + 按 `TLV` 组织的信息单元区”框架；`信息单元类型`、`长度类型`、`站点属性信息`、`站点路由信息`、`邻居节点信道信息` 的组织方式均未变化，主要调整是规范归属从 `1.0` 双模无线通信接入协议迁移到 `2.0` 数据链路层通信协议，同时表号改为 `139-152`。

| 对比项 | 1.0 版 | 2.0 版 | 结论 |
| --- | --- | --- | --- |
| 章节归属 | `8.1.4.18`，表 `214-227`，归属于 `1.0` 双模无线通信接入协议 | `5.1.4.18`，表 `139-152`，归属于 `2.0` 数据链路层通信协议 | 主要是规范编排迁移 |
| 报文外层布局 | `MAC(6) + 统计序号(1) + N 个信息单元`；信息单元头为 `7 bit` 类型 + `1 bit` 长度类型 + `1/2` 字节长度 + 内容 | 与 `1.0` 相同 | 外层骨架未变 |
| 信息单元类型与长度类型 | 类型 `0-3` 仍为站点属性、站点路由、邻居节点信道非位图、邻居节点信道位图；长度类型 `0/1` 对应 `1/2` 字节长度字段 | 与 `1.0` 相同 | `TLV` 定义未变 |
| 站点属性与站点路由信息 | 表 `217/218` 定义 `14` 字节站点属性信息和 `2` 字节/项的路由条目，字段顺序包括 `CCO MAC`、`代理 TEI`、`角色`、`层级`、`链路 RF 跳数`、成功率、周期和老化个数 | 表 `142/143` 定义一致 | 固定格式信息单元未变 |
| 邻居节点信道信息 | 表 `219-225`、表 `226-227` 定义非位图/位图两类邻居节点信道信息，组合类型、`TEI` 长度、接收率、`SNR`、`RSSI` 长度及映射规则均已给出 | 表 `144-150`、表 `151-152` 延续同一套定义 | 复杂可变信息单元未见新增字段或删改 |

整体上，`2.0` 版 `MMeRFDiscoverNodeList` 可以视为对 `1.0` 双模定义的直接承接，属于规范迁移与重编号，而不是报文格式重设计。

## 校准说明

- 原 PDF 为扫描件，`pdftotext` 对本节几乎只抽出水印文本，页图是主要依据。
- `MMeRfDiscoverNodeList_t`、`rf_infounit_1byte_hdr_t`、`rf_infounit_2byte_hdr_t`、`sta_attr_info_t`、`sta_route_info_t` 位于 `h2-link-v2-sdk/SDK/src/spg/mac/mmsg.h`，用于交叉确认固定头和信息单元头布局。
- 表 `142` 的最后两行在下一页续表中，故站点属性信息长度校准为 `14` 字节。
- `消息类型 0x0071` 是推定值，不是 SDK 中现成枚举值。
  - 依据 1：上一条已定义的无线信道冲突上报报文在 SDK 中是 `0x0070`
  - 依据 2：本节在协议章节顺序上紧随其后
  - 依据 3：SDK 已存在 `MMeRfDiscoverNodeList_t` 结构，但缺少对应枚举，说明枚举表尚未补齐
