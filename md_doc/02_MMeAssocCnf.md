### 表 93 关联请求的确认结果

| 值 | 定义 |
| --- | --- |
| 0x00 | 表示关联请求成功 |
| 0x01 | 表示该站点不在白名单中 |
| 0x02 | 保留 |
| 0x03 | 表示加入的站点个数超过上限 |
| 0x04 | 表示没有设置白名单列表 |
| 0x05 | 表示代理站点个数超过上限 |
| 0x06 | 表示子站点个数超过上限 |
| 0x07 | 保留 |
| 0x08 | 表示重复的 MAC 地址 |
| 0x09 | 表示超过拓扑层级 |
| 0x0A | 表示站点再次关联请求入网成功 |
| 0x0B | 表示新的站点试图以自己的子站点为代理来入网 |
| 0x0C | 表示组网拓扑中存在环路 |
| 0x0D | 表示 CCO 端未知原因出错 |

| 字段 | 字节号 | 字段大小（字节） | 说明 |

| 路由表大小 | 4-5 | 2 | 路由信息表大小，取值为路由表总长度/2 |

## 实现说明

- `02_MMeAssocCnf.h/.c` 按表 92 对固定头做了完整编解码。
- `站点信息` 按表 97 拆成结构体头和原始 `RouteTable` 字节流，避免在 demo 中引入动态内存。
- `RouteTable` 当前按表 98 原样保留，调用方可基于 `DirectStaCount`、`DirectProxyCount` 和 `RouteTableSize` 继续细分解析。
- 解码函数不会拷贝 `RouteTable`，`Message->StationInfo.RouteTable` 会直接指向输入缓冲区中的对应位置。

## 与 1.0 版本对比

对比基准为 `doc/南网hplc+hrf双模模块1.0标准规范/2、计量自动化系统双模通信技术规范/《南方电网计量自动化系统双模通信技术规范 第2部分：无线通信接入协议》（报批稿）.pdf` 的 `8.1.4.2 关联确认报文`（表 `170` 至表 `176`）。

从报文整体布局看，`2.0` 版 `MMeAssocCnf` 仍保持 `1.0` 的“固定 `36` 字节头 + 字节 `36` 起可变尾部信息”框架，`站点 MAC 地址`、`结果`、`层级`、`TEI`、`随机数`、`重新关联时间`、`端到端序列号`、`路径序号`、`组网序列号` 和 `管理消息版本` 的位置均未变化；实质变化主要集中在字节 `15` 的 `载波频段` 位宽扩展，以及尾部可变区的命名和说明方式调整。

| 对比项 | 1.0 版 | 2.0 版 | 结论 |
| --- | --- | --- | --- |
| 报文骨架 | 固定头到字节 `35`，字节 `36` 起为 `路由表信息` 可变区 | 固定头到字节 `35`，字节 `36` 起为 `站点信息` 可变区 | 外层布局未变 |
| 字节 `15` 位域 | `bit0` 为 `链路类型`，`bit1-bit2` 为 `载波频段`，其余 `5 bit` 保留 | `bit0` 为 `链路类型`，`bit1-bit3` 为 `载波频段`，其余 `4 bit` 保留 | `2.0` 扩大了频段字段位宽，占用了 `1` 个原保留位 |
| 载波频段表达能力 | 表 `173` 仅给出频段 `0-2` | 本文件按 `2.0` 页图校准为 `3 bit` 频段字段，可覆盖比 `1.0` 更多的频段编码值 | `2.0` 为新增频段预留了编码空间 |
| 尾部可变信息定义 | 名称为 `路由表信息`，表 `175` 由 `直连站点数`、`直连代理数`、`路由表大小`、`保留`、`路由表` 组成 | 名称改为 `站点信息`，表 `97` 内部字段仍为 `直连站点数`、`直连代理数`、`路由表大小`、`保留`、`路由表` | 主要是命名和章节组织调整，线缆格式基本兼容 |
| 结果码与其余固定字段 | `结果`、`层级`、`TEI`、`随机数`、`重新关联时间`、`端到端序列号`、`路径序号`、`组网序列号`、`探测频段标识符` 均已定义 | 上述字段在 `2.0` 中继续保留，取值语义未见实质变化 | 主体语义延续 `1.0` |

整体上，`2.0` 版 `MMeAssocCnf` 属于在 `1.0` 报文框架上的兼容演进，而不是报文重设计。
