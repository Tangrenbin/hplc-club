## 实现说明

- `17_MMeRFChannelConflictReport.h/.c` 按“固定头 `7` 字节 + 邻居条目列表”的形式实现了完整编解码。
- 每个邻居条目使用原始 `2` 字节结构 `MMeRFChannelConflictReportItem_t` 表示：
- 第 `1` 个字节保存 `无线信道号`
- 第 `2` 个字节保存 `option + 保留位`
- 提供 `mme_rf_channel_conflict_report_item_get_wireless_option` / `mme_rf_channel_conflict_report_item_set_wireless_option` helper 读写条目中的低 `2 bit option`。
- `validate` 当前做两类校验：
- 当 `NeighborNetworkCount > 0` 时，`NeighborNetworkList` 不能为空
- 每个条目的保留位 `bit[7:2]` 必须为 `0`
- 规范页未给出 `无线信道号` 的取值范围，因此实现将其按原始 `uint8_t` 字段保留，不额外收紧校验。
- 规范页未给出 `邻居网络个数` 的最小值，因此实现允许个数为 `0`，此时报文只有固定 `7` 字节头。
- 解码成功后，`Message->NeighborNetworkList` 直接指向输入缓冲区中的邻居条目列表，不额外拷贝。

## 与 1.0 版本对比

对比基准为 `doc/南网hplc+hrf双模模块1.0标准规范/2、计量自动化系统双模通信技术规范/《南方电网计量自动化系统双模通信技术规范 第2部分：无线通信接入协议》（报批稿）.pdf` 的 `8.1.4.17 无线信道冲突上报报文`（表 `212` 至表 `213`）。

从报文整体布局看，`2.0` 版 `MMeRFChannelConflictReport` 与 `1.0` 版保持实质一致，仍为“`6` 字节冲突网络 `CCO MAC` 地址 + `1` 字节邻居网络个数 + 可变长邻居网络条目列表”的结构；每个邻居条目也仍按 `2` 字节交错组织，未见新增字段、删减字段或保留位重新定义。

| 对比项 | 1.0 版 | 2.0 版 | 结论 |
| --- | --- | --- | --- |
| 章节归属 | `8.1.4.17`，表 `212-213`，归属于 `1.0` 双模无线通信接入协议 | `5.1.4.17`，表 `137-138`，归属于 `2.0` 数据链路层通信协议 | 规范迁移并重编号 |
| 报文主体骨架 | 固定头为 `CCO MAC 地址(6B) + 邻居网络个数(1B)`，后接 `L` 字节邻居网络条目 | 与 `1.0` 相同，固定头仍为 `7` 字节，尾随可变长列表 | 外层布局未变 |
| 邻居条目编码 | 每个邻居占 `2` 字节：前 `1` 字节为 `无线信道号`，后 `1` 字节低 `2 bit` 为 `无线 option`、高 `6 bit` 保留 | 与 `1.0` 相同，条目重复模式和字节位置未变 | 条目编码未变 |
| 保留位语义 | `option` 所在字节的其余 `6 bit` 标记为保留 | 与 `1.0` 相同，当前 `2.0` 校准内容未见新增语义 | 位域定义未变 |
| 可变长列表表达 | `邻居网络个数` 描述周边可见邻居数量，列表按条目数顺序展开 | 与 `1.0` 相同，仍由个数字段配合变长条目列表表达 | 变长规则未变 |

整体上，`2.0` 版 `MMeRFChannelConflictReport` 可以视为对 `1.0` 定义的直接承接，属于规范迁移与表号调整，不是报文格式重设计。

## 校准说明

- 第 `62` 页包含章节标题、表 137 以及表 138 的前半部分；第 `63` 页顶部继续给出了表 138 的重复条目模式。
- 原 PDF 为扫描版，`pdftotext` 抽取结果不可直接使用，本文件基于第 `62-63` 页页图人工校准。
- 消息类型 `0x0070` 与固定头布局 `cco_mac + nr_neighbor + data[0]` 使用 `h2-link-v2-sdk/SDK/src/spg/mac/mmsg.h` 中的 `MMeRFChannelConflictReport_t` 交叉确认。
- 表 138 明确给出了“信道号/option/保留”按条目交错排列的 2 字节模式，因此本次实现采用交错编码。
- `1.0` 双模规范 `8.1.4.17` 的表 `212/213` 与 `2.0` 表 `137/138` 在条目布局上保持一致；仓库旧代码 `h2-link-v1-code/SDK/src/datalink/dl_mgm_msg_spg.c` 里单邻居发送路径使用 `NBnetList[1] = option & 0x03`，可辅助确认 `option` 位于低 `2 bit`，但其多邻居内存组织与 `1.0/2.0` 规范表格均不一致，本次实现不沿用旧版列表布局。
