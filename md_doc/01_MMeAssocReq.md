#### 5.1.4.1.1 站点 MAC 地址

表示发起关联请求的终端设备地址，长度为 6 字节。

#### 5.1.4.1.2 代理 TEI

包含候选代理站点列表，最多支持携带 5 个候选代理站点的 TEI。

#### 5.1.4.1.3 链路类型

表示发送关联请求站点与候选代理的通信链路类型，链路类型值定义如表 79 所示。

| 值 | 定义 |
| --- | --- |
| 0 | 高速载波链路 |
| 1 | 无线链路 |

#### 5.1.4.1.4 相线

表示本站点所属相线的评估结果。最低字节存放评估出的所属相线，其他字节依次填入可能的备选相位，相线值定义如表 80 所示。

| 值 | 定义 |
| --- | --- |
| 0 | 表示未知相线 |
| 1 | 表示 A 相线 |
| 2 | 表示 B 相线 |
| 3 | 表示 C 相线 |

#### 5.1.4.1.5 设备类型

表示设备类型，定义如表 81 所示。

| 值 | 定义 |
| --- | --- |
| 0x01 | 抄控器 |
| 0x02 | 集中器通信模块 |
| 0x03 | 单相电表通信模块 |
| 0x04 | 中继器 |
| 0x05 | II 型采集器 |
| 0x06 | I 型采集器 |
| 0x07 | 三相表通信模块 |
| 0x08-0xFF | 保留 |

#### 5.1.4.1.6 扩展特性版本号

表示节点支持的扩展特性，定义如表 82 所示。

| 值 | 定义 |
| --- | --- |
| 其它 | 保留 |
| 0x1 | 指示模块支持应用层多帧抄读 |

#### 5.1.4.1.7 MAC 地址类型

表示关联入网时使用的 MAC 地址来源，定义如表 83 所示。

| 值 | 定义 |
| --- | --- |
| 0x0 | 电能表地址作为入网 MAC 地址，应用于电表场景 |
| 0x1 | 通信模块本身的 MAC 地址作为入网 MAC 地址，应用于其他异常场景，如获取电能表地址失败 |
| 0x2 | 采集器地址作为入网 MAC 地址，应用于 I 型采集器及 II 型采集器场景 |

#### 5.1.4.1.8 模块类型

表示发送关联请求站点通信模块的类型，定义如表 84 所示。

| 值 | 定义 |
| --- | --- |
| 0 | HPLC 单模模块 |
| 1 | 双模 HPLC 和 RF 模块 |
| 2 | 无线单模模块 |

#### 5.1.4.1.9 站点关联随机数

表示模块关联入网的随机数。设备出厂后，初次上电时会自动获取一个 32 比特的随机值作为关联随机数，后续掉电后再上电，不再重新获取。

## 与 1.0 版本对比

对比基准为 `doc/南网hplc+hrf双模模块1.0标准规范/2、计量自动化系统双模通信技术规范/《南方电网计量自动化系统双模通信技术规范 第2部分：无线通信接入协议》（报批稿）.pdf` 的 `8.1.4.1 关联请求报文`（表 `159` 至表 `169`）。

从报文整体布局看，`2.0` 版 `MMeAssocReq` 仍保持 `1.0` 的固定 `68` 字节结构，`站点 MAC 地址`、`候选代理 TEI`、`相线`、`随机数`、`版本信息`、`复位次数`、`代理类型`、`组网序列号` 和 `端到端序列号` 的位置均未变化；实质差异集中在原保留字节和保留位的重新定义，用来补充节点扩展能力和新一代载波能力描述。

| 对比项 | 1.0 版 | 2.0 版 | 结论 |
| --- | --- | --- | --- |
| 报文总长与主体骨架 | 表 `159` 定义为固定 `68` 字节，请求帧主体字段分布到字节 `0-67` | 与 `1.0` 相同，仍为固定 `68` 字节 | 外层布局未变 |
| 字节 `21` 定义 | `1` 字节保留 | 重新定义为 `扩展特性版本号`，用于声明节点支持的扩展特性 | `2.0` 在原保留字节上增加能力协商信息 |
| 字节 `62` 位域 | `bit0` 保留，`bit1-bit4` 为 `管理消息版本`，`bit5-bit7` 保留 | `bit0` 仍保留，`bit1-bit4` 仍为 `管理消息版本`，新增 `PLC 能力指示` 和 `通道数` | `2.0` 扩展了节点物理层能力表达 |
| 支持频段标识语义 | `bit0-bit1` 定义频段；`0x0` 表示频段 `0/1`，`0x1` 表示频段 `0/1/2` | 仍位于字节 `63` 的 `bit0-bit1`，但 `0x1` 在 `PLC 能力指示 = 1` 时可扩展表示频段 `0-4` | 频段字段位置不变，语义随新增能力位扩展 |
| 其余字段定义 | `MAC 地址类型`、`模块类型`、`版本信息`、`代理类型` 等已给出完整定义 | 上述字段定义整体延续 `1.0` | 兼容字段保持不变 |

整体上，`2.0` 版 `MMeAssocReq` 属于在 `1.0` 固定帧基础上的兼容扩展，而不是报文重设计。
