### 5.1.4.14 过零NTB采集指示报文

过零NTB采集指示报文（`MMeZeroCrossNTBCollectInd`）格式定义如表 130 所示。

规范原文说明：本报文由 CCO 创建发送。

#### 表 130 过零NTB采集指示报文格式

| 字段 | 字节号 | 字段大小(字节) | 说明 |
| --- | --- | --- | --- |
| 站点TEI | 0-1 | 2 | 需要进行过零NTB采集的站点TEI |
| 站点类型 | 2 | 1 | 见表 131 |
| 采集周期 | 3 | 1 | 见表 132 |
| 采集数量 | 4 | 1 | 需要采集的过零NTB值数量 |

#### 5.1.4.14.1 站点TEI

- 表示需要进行过零NTB采集的站点TEI。
- 站点TEI只在“站点类型”为单站点采集时有效。
- 当站点类型为全网站点时，站点TEI填写为 `0xFFFF`。

#### 5.1.4.14.2 站点类型

站点类型定义如表 131 所示。

#### 表 131 过零NTB采集站点类型

| 值 | 定义 |
| --- | --- |
| `0` | 单站点 |
| `1` | 全网站点 |
| 其他 | 保留 |

#### 5.1.4.14.3 采集周期

采集周期定义如表 132 所示。

#### 表 132 过零NTB采集周期

| 值 | 定义 |
| --- | --- |
| `0` | 半个电力线周期 |
| `1` | 一个电力线周期 |
| 其他 | 保留 |

注：在 50Hz 的电力线上，一个电力线周期是一个正弦波周期 20ms。采集的过零NTB时刻，与理论过零时刻的偏差必须保证在 `+/-1.2ms` 以内。半个电力线周期表示每次过零均需采集 NTB 值，一个电力线周期表示每两次过零采集一次 NTB 值。

#### 5.1.4.14.4 采集数量

- 表示需要采集过零NTB值的数量。
- 指示报文下发后，所指定站点需要连续采集指定数量的过零点NTB值。

## 实现说明

- `14_MMeZeroCrossNTBCollectInd.h/.c` 按表 130 实现固定 5 字节报文的编解码。
- `站点TEI` 按 SDK 结构体保留为原始 `uint16_t` 小端字段，不额外拆分 bit 位。
- `采集数量` 按原始 `uint8_t` 字段保留，规范页未给出取值范围，因此实现未额外增加上下界限制。
- `validate` 会校验：
- `站点类型` 仅允许 `0` 或 `1`
- `采集周期` 仅允许 `0` 或 `1`
- `站点类型 = 全网站点` 时，`站点TEI` 必须为 `0xFFFF`
- `站点类型 = 单站点` 时，`站点TEI` 不能为 `0xFFFF`
- `init` 默认初始化为“全网站点 + 半个电力线周期 + 采集 1 次”，这是为了提供一个可直接编码的默认示例值。

## 与 1.0 版本对比

对比基准为 `doc/南网hplc+hrf双模模块1.0标准规范/2、计量自动化系统双模通信技术规范/《南方电网计量自动化系统双模通信技术规范 第2部分：无线通信接入协议》（报批稿）.pdf` 的 `8.1.4.14 过零 NTB 采集指示报文`（表 `205` 至表 `207`）。

从报文整体布局看，`2.0` 版 `MMeZeroCrossNTBCollectInd` 与 `1.0` 版保持实质一致，固定长度仍为 `5` 字节，字段顺序仍为 `站点TEI + 站点类型 + 采集周期 + 采集数量`；`站点TEI = 0xFFFF` 表示全网站点、`站点类型` 与 `采集周期` 的枚举值，以及过零采集偏差需控制在 `+/-1.2ms` 以内的约束都未见变化，主要差异仅是章节号和表号迁移到 `2.0` 规范体系。

| 对比项 | 1.0 版 | 2.0 版 | 结论 |
| --- | --- | --- | --- |
| 章节归属 | `8.1.4.14`，表 `205-207`，归属于 `1.0` 双模无线通信接入协议 | `5.1.4.14`，表 `130-132`，归属于 `2.0` 数据链路层通信协议 | 主要是规范迁移与重编号 |
| 报文总长与字段顺序 | 固定 `5` 字节，字段顺序为 `站点TEI`、`站点类型`、`采集周期`、`采集数量` | 与 `1.0` 相同 | 外层布局未变 |
| `站点TEI` 与站点范围语义 | 单站点采集时有效；全网站点时填写 `0xFFFF` | 与 `1.0` 相同 | 目标站点表达方式未变 |
| `站点类型` 定义 | `0 = 单站点`，`1 = 全网站点`，其他保留 | 与 `1.0` 相同 | 枚举值未变 |
| `采集周期` 定义 | `0 = 半个电力线周期`，`1 = 一个电力线周期`，其他保留；偏差要求 `+/-1.2ms` | 与 `1.0` 相同 | 周期编码和精度要求未变 |
| `采集数量` 语义 | 表示连续采集的过零 NTB 数量 | 与 `1.0` 相同 | 业务含义未变 |

整体上，`2.0` 版 `MMeZeroCrossNTBCollectInd` 属于对 `1.0` 定义的直接承接，主要是章节和表号调整，而不是报文格式重设计。
